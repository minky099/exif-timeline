# 사진 촬영시각 순차 기입 웹앱

메신저(카카오톡 등)로 받은 사진은 재압축되며 EXIF가 제거된다. iOS 사진앱은
EXIF `DateTimeOriginal`이 있으면 그 시각을 자산 촬영일로 쓰지만, 없으면
**다운로드 시각**으로 잡는다. 그래서 EXIF 없는 사진을 NAS 자동백업(Immich/Synology)하면
전부 다운로드 시각으로 몰려 순서가 뒤죽박죽이 된다.

이 웹앱은 **자동백업이 돌기 전에** 받은 사진들에 촬영시각을 순서대로 서로 다르게
EXIF로 기입한다. 아이폰 Safari에서 열어 쓴다. 서버 없이 기기 안에서만 처리한다.

- `index.html` — 본 앱 (여러 장 일괄 처리)
- `exif-test.html` — 실기기 최소 검증 (1장)

## ⚠️ 먼저 실기기 검증부터

**아직 검증되지 않은 핵심 가정:** "piexifjs로 `DateTimeOriginal`을 써넣은 이미지를
iOS 사진앱에 추가하면, 사진앱이 그 시각을 자산 촬영일로 인식한다."

저장 방식에 따라 iOS가 EXIF를 읽을 수도, **저장 시각으로 덮을 수도** 있다.
본 앱을 여러 장에 쓰기 전에 반드시 `exif-test.html`로 **1장 검증**을 먼저 하라.

1. `exif-test.html`을 아이폰 Safari에서 연다
2. JPEG 1장 선택 → 임의 촬영시각 입력 → **EXIF 기입하기**
3. 저장(아래 "저장 방식" 참고)
4. 사진앱에서 방금 저장한 사진의 **정보(ⓘ)** 확인 → 날짜/시각이 기입값과 같으면 **검증 성공**
5. 다르면(저장 시각으로 덮임) → "저장 방식"을 바꿔 재시도. 그래도 안 되면 아래 **대안** 참고

## 사용 순서 (자동백업 전에!)

> Immich/Synology 자동백업이 앨범을 훑기 **전에** 이 앱으로 처리해야 한다.
> 이미 백업된 사진은 이 앱으로 촬영일을 고쳐도 재백업 트리거가 안 될 수 있다.

1. 아이폰 Safari로 `index.html`을 연다 (호스팅 방법은 아래 참고)
2. **① 사진 선택** — 받은 JPEG 여러 장 추가 (HEIC는 자동 제외되고 안내됨)
3. **③ 순서 & 묶음** — ≡ 손잡이를 끌어 순서를 맞추고, 썸네일의 **✂ 나누기**로 묶음 경계를 넣는다
   (묶음 머리의 **⭱ 위와 합치기**로 경계 제거)
4. **② 시간 범위** — 첫 사진 시각 / 마지막 사진 시각을 초 단위로 입력, **묶음 사이 벌어짐** 슬라이더 조절
5. 각 썸네일에 배정될 시각이 미리보기로 표시된다 — 확인
6. **EXIF 기입하기** → **사진에 저장(공유시트)** 으로 앨범에 저장
7. 자동백업이 돌면 촬영시각 순서대로 정렬된다

### 아이폰에서 여는 방법
- **Docker (권장 · NAS 상시 구동):** 아래 "Docker 배포" 참고. `http://<NAS-IP>:9003` 로 접속.
- **GitHub Pages:** 저장소 Settings › Pages › Branch를 `main`으로 배포하면
  `https://<user>.github.io/exif-timeline/` 로 접속. HTTPS라 공유시트 저장까지 잘 된다.
- **로컬 파일:** `index.html`을 파일앱에 저장 후 Safari로 열어도 동작한다(오프라인 가능).
  라이브러리를 인라인해 둬서 네트워크 없이 작동한다.

## Docker 배포

단일 HTML을 nginx로 정적 서빙한다. 빌드 단계는 없다. 포트는 **9003**
(EXIF `DateTimeOriginal` 태그 ID `0x9003`에서 따옴).

```bash
docker compose up -d --build
# → http://<호스트-IP>:9003
```
직접 빌드/실행:
```bash
docker build -t exif-timeline .
docker run -d --name exif-timeline -p 9003:9003 --restart unless-stopped exif-timeline
```
- 헬스체크: `GET /healthz` → `ok`
- Synology(DSM): Container Manager에서 이 저장소로 프로젝트 생성 → `docker-compose.yml` 사용,
  로컬 포트 `9003`(원하면 다른 포트로 매핑)로 노출. Immich와 같은 NAS에 함께 올려두면 편하다.

> **공유시트 저장(navigator.share)은 보안 컨텍스트에서만 동작한다.** `http://<LAN-IP>:9003`
> 같은 평문 HTTP로 접속하면 iOS가 파일 공유 API를 막아, 앱은 자동으로 **롱프레스 "사진에 추가" /
> 다운로드** 폴백으로 전환된다(정상 동작). 공유시트까지 쓰려면 `localhost` 또는 **HTTPS**로 접속하라
> — 예: Synology 리버스 프록시로 `https://` 도메인을 붙이면 공유시트 저장이 활성화된다.

## 저장 방식 (iOS)

우선순위대로 시도한다. **실제 이미지 파일을 그대로 넘기는 방식**이라야 EXIF가 보존된다.

1. **공유시트 (기본):** "사진에 저장(공유시트 열기)" → 시트에서 **이미지 저장** 선택.
   여러 장을 한 번에 앨범에 넣는다. (`navigator.share({files})` 사용)
2. **미리보기 롱프레스:** 결과 이미지를 **길게 눌러 "사진에 추가"**.
3. **파일앱 경유:** 각 이미지 다운로드 → 파일앱 → 사진앱으로 가져오기.

### 가정이 깨질 때의 대안
저장한 사진의 촬영일이 여전히 저장 시각으로 잡힌다면:
- **저장 방식을 바꿔본다** (위 1 → 2 → 3 순서로). 기기/iOS 버전에 따라 EXIF를 읽는 경로가 다르다.
- **최후 수단 — 메타데이터 편집 앱 경유:** 이 앱으로 EXIF를 넣은 JPEG를 저장한 뒤,
  [Metapho](https://apps.apple.com/app/metapho/id914457352) 같은 앱으로 열어
  촬영일을 확정 적용한다. Metapho는 사진앱 자산의 날짜를 직접 고칠 수 있어 가장 확실한 대비책이다.

## 시간 분배 로직 (그룹 가중치)

사진을 **묶음(그룹)** 으로 지정한다. 묶음 안은 촘촘, 묶음 사이는 넓게.

- 그룹 수 `G`, 각 그룹 사진 수 `k_g`, 전체 `N = Σk_g`
- 내부 간격 개수 `= Σ(k_g − 1)`, 경계 간격 개수 `= G − 1`
- 내부 가중치 `w_in`(=1), 경계 가중치 `w_bd`(기본 8, 슬라이더로 조절)
- 가중치총합 `= Σ(k_g−1)·w_in + (G−1)·w_bd`
- 단위초 `= (E − S) / 가중치총합`
- 첫 사진 `= 시작시각 S`, 마지막 사진 `= 끝시각 E` 를 정확히 유지.
  이후 각 사진 `= 앞 사진 + (내부면 단위초·w_in, 경계면 단위초·w_bd)`.
- `N == 1` 이면 시작시각 하나만 배정(나눗셈 0 방지).

**검증 예:** 3그룹 `[3, 2, 4]`장, 14:00:00 - 14:30:00, `w_bd = 8`
```
14:00:00  14:01:21  14:02:43  |  14:13:38  14:15:00  |  14:25:54  14:27:16  14:28:38  14:30:00
```
(`|` = 묶음 경계)

## 제약 / 아키텍처

- **단일 HTML 파일, 클라이언트 전용.** 서버·빌드 없음. 사진은 기기 밖으로 나가지 않는다.
- **JPEG 전용.** EXIF 기입에 [piexifjs](https://github.com/hMatoba/piexifjs) 1.0.6 사용.
  piexifjs는 HEIC/HEIF를 지원하지 않으므로 HEIC 입력은 매직바이트로 감지해 제외하고 안내한다.
- 기입 필드: `DateTimeOriginal`(0x9003) + `DateTimeDigitized`(0x9004) + 0th `DateTime`(0x0132).
  기존 EXIF(방향 태그 등)는 최대한 보존하며, 재기입 실패 시 촬영시각+방향만 남기는 최소 구성으로 대체한다.
- 라이브러리는 각 HTML에 **인라인**되어 있다. CDN/네트워크 불필요, 오프라인 동작.
  로드 실패 시 침묵하지 않고 명확한 경고를 띄운다.

### piexifjs 인라인 재빌드
`index.html` / `exif-test.html`의 첫 번째 `<script>` 블록이 piexifjs 1.0.6(`piexif.js`, `piexif.min.js` 아님)의
전체 소스다. 갱신하려면:
```bash
npm pack piexifjs@1.0.6           # piexifjs-1.0.6.tgz → package/piexif.js
# 각 HTML의 첫 <script>…</script> 안 내용을 package/piexif.js 로 교체
```

### 표기 규칙
- 숫자 범위는 물결(`~`)이 아니라 하이픈(`-`)으로 쓴다. 예: `14:00 - 14:30`.
